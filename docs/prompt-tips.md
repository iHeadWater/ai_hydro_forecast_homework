# Prompt 编写技巧

> 好 Prompt 是高效使用 AI Agent 的关键。本章教你怎么写、怎么写得好。

## 核心原则

### 1. 明确说你要什么，不要什么

| ❌ 差的 Prompt | ✅ 好的 Prompt |
|---------------|---------------|
| "帮我做个水文分析" | "帮我加载 CAMELS 数据集里的 01022500 号流域，时间范围 2000-2010，用 XAJ 模型做率定" |
| "画个图" | "把率定期的降雨-径流过程图画出来，上面降雨下面流量，标注 NSE 值" |
| "模型跑一下" | "用 SCE-UA 算法率定 XAJ 模型，目标函数用 NSE，最大迭代 1000 次" |

### 2. 给上下文，不要假设 AI 什么都知道

```
"这个项目用的是 hydromodel 包里的模型，
数据通过 hydrodatasource 加载。
参考 CLAUDE.md 里的 API 来操作。"
```

### 3. 分步骤，别一口气全说

不要一次说：
> "帮我加载数据做率定然后评估然后画图"

而是拆成多轮对话：
1. "帮我加载流域 X 的数据，看看有哪些字段"
2. "好的，现在用 XAJ 模型做率定"
3. "现在计算 NSE 和 KGE"
4. "最后画个降雨径流过程图"

### 4. 让 AI 先解释再执行

加上这句话可以大幅提高成功率：
> "在写代码之前，先告诉我你的理解，我确认后再执行"

## 水文领域常用术语

在 Prompt 里用这些词，AI 更容易理解你的需求：

| 类别 | 术语 |
|------|------|
| 模型 | XAJ（新安江）、GR4J、GR6J、HYMOD、LSTM、Seq2Seq |
| 数据 | CAMELS、ERA5-Land、NLDAS、precipitation（降水）、streamflow（径流）、PET（潜在蒸散发） |
| 率定 | SCE-UA、Genetic Algorithm（遗传算法）、Differential Evolution（差分进化） |
| 指标 | NSE（纳什效率系数）、KGE、RMSE、MAE、runoff coefficient（径流系数） |
| 阶段 | warmup（预热期）、calibration period（率定期）、validation period（验证期） |

## 让 AI 自我纠错

善用这些指令让 AI 自己检查：

- "运行这段代码，如果有错误就自己修"
- "你确定这个结果是合理的吗？再检查一遍"
- "NSE 大于 1 是不可能的，请检查哪里出了问题"
- "把刚才报的错修好，然后重新运行"

## 常见翻车现场

### 1. 单位搞错

**症状**：NSE 跑出来是负数或者特别小
**解决**：告诉 AI "检查一下单位，可能需要 mm/day 转 m³/s"

### 2. 忘记去掉预热期

**症状**：前几十个时间步的模拟值和观测差很多
**解决**："预热期的数据要去掉再算指标，warmup_length 设为 365"

### 3. 数据没对齐

**症状**：报错说时间索引不匹配
**解决**："检查一下降水和径流的时间范围是否一致"

### 4. 参数超出范围

**症状**：模型运行报错或结果异常
**解决**："参数需要在 param_range 定义的范围内，检查一下参数值"

## 进阶技巧

### 让 AI 扮演专家角色

```
"你现在是一个有 10 年经验的水文建模专家。
请帮我做以下分析..."
```

### 提供参考标准

```
"Moriasi et al. (2007) 的标准是 NSE > 0.5 为满意。
我的率定结果 NSE 只有 0.3，帮我分析可能的原因和改进方向。"
```

### 要求输出格式

```
"结果用表格展示，包含：参数名、率定值、参数范围。
然后附一段 100 字以内的总结。"
```
